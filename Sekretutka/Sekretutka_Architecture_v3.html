Sekretutka Architecture v5

Этот документ сейчас вторичен по отношению к `PRD + README + CRM`. Целевая модель: `GAME_RULES` хранит реестр правил и player-authored declarations, `GAME_LOG` хранит историю событий, а `PROFILE_GAME` остается гибридным листом goals + decomposition + snapshot + runtime block.

Rules
`GAME_RULES` = мир игры Системные законы, option sets, transition rules, attempt rules и выбранные игроком декларации лежат в одном месте.
History
`GAME_LOG` = история Все изменения, награды, дебаффы, попытки и player-declaration events пишутся в единый журнал.
Snapshot
`PROFILE_GAME` = goals + snapshot Канонически это гибридный лист: top goals, decomposition zone, актуальные значения игрока, readiness-формулы и нижний технический блок для workflow.
Views
`Dashboard` и planners = витрины Task analytics, daily planner и weekly planner показывают данные, но не становятся источниками правды.

Что живёт только в `PROFILE_GAME`

  • top goals и decomposition zone;
  • текущие активные значения игрока;
  • формулы readiness и прогрессии;
  • живой snapshot stage / level / risk / sprint;
  • нижний технический блок для engine и n8n.

Что больше не нужно дублировать в `PROFILE_GAME`

  • task counters и dashboard analytics;
  • today / upcoming / completed lists;
  • status / category / priority breakdown;
  • отдельный лист `STAGE_RULES`.

Игрок сам пишет себе правила игры

Во время онбординга и дальнейшей настройки игрок создаёт не просто профильные значения, а player-authored declarations в GAME_RULES. Snapshot `PROFILE_GAME` собирается из формул и runtime-логики, а история изменений живет в GAME_LOG.

Player declarations в `GAME_RULES`

Система хранит в одном месте:

  • TREASURE
  • выбранный TREASURE_TYPE
  • выбранный RISK_MODE
  • тактические player-authored declarations и level naming rules
  • выбранный SPRINT_DAYS
  • выбранный TREASURE_HORIZON_DAYS
  • рамки и правила ATTEMPTS

Основные законы игры

Level = накопленный прирост запаса +1 000 000 THB с момента LEVEL_START_TS Player Status = XP_TOTAL TOTAL_PROGRESS = 0.4×(Money_Growth/1 000 000) + 0.3×(Done_Tasks/90) + 0.2×(XP_from_level_start/5000) + 0.1×(Beliefs_Count/3) LEVEL_READY = TRUE, если TOTAL_PROGRESS >= 1.0 и запас THB >= 1 000 000

Bio-Layer

Это слой HP, цикла и recovery-режимов. Он накладывается поверх Game-layer: при низком HP нельзя просто “взять себя в руки”, а работа и сама игра возможны только при энергии.

Stage

Stage не назначается игроком. Система автоматически считает его внутри текущего attempt по прогрессу `XP_TOTAL` от точки старта attempt.

Attempts

Attempt — это период консистентного присутствия в игре. Он стартует после онбординга, падает после 5 дней тишины и полностью фиксируется через `GAME_LOG`.

Recovery Modes

  • NORMAL: `HP >= 100`
  • RECOVERY_ACTIVE: `HP < 60`
  • RECOVERY_PASSIVE: `HP < 30`

Task Tracks

  • Craft: деньги, опыт, дело, навык
  • Social: отношения, поддержка, влияние
  • Vitality: тело, здоровье, восстановление

Onboarding Order

После промо идут `ONBOARDING_INTRO`, bio/reality-калибровка, спринт, задачи по трекам, привычки, KPI, цель, боссы, recovery-rules, shop и только потом контракт.

Как это работает в runtime

Workflow Что читает Что пишет
Main `PROFILE_GAME`, task context ничего напрямую; только маршрутизирует и честно возвращает результат
Quest Architect `PROFILE_GAME`, `Task List`, часть `GAME_RULES` по необходимости `Task List`, `GAME_LOG`, daily sheets
Game Rules / onboarding `GAME_RULES`, `GAME_LOG`, `PROFILE_GAME` runtime block `GAME_RULES`, `GAME_LOG`
ENGINE `PROFILE_GAME`, `GAME_LOG`, `Task List`, planners runtime block в `PROFILE_GAME`, сообщения, системные обновления, weekly setup

Когда меняется player declaration

  • workflow обновляет/создаёт запись в GAME_RULES;
  • обновляет текущее значение в PROFILE_GAME только если это runtime snapshot, а не player-authored rule;
  • добавляет событие в GAME_LOG.

Какие события фиксируются в `GAME_LOG`

  • TREASURE_SET
  • TREASURE_UPDATED
  • SPRINT_DAYS_CHANGED
  • RISK_MODE_CHANGED
  • PLAYER_STAGE_CHANGED
  • LEVEL_NAME_CHANGED
  • ATTEMPT_STARTED
  • ATTEMPT_FINISHED
